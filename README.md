# Projet SQL - cIAra Mobility

## Présentation
Dans ce projet, nous avons travaillé sur une base de données fournie par l’entreprise **cIAra Mobility**, spécialisée dans la location de véhicules électriques partagés en France. L’objectif était d’exploiter cette base de données à l’aide du langage SQL afin de répondre à différents besoins métier. Nous devions interroger les données, les analyser et construire des requêtes adaptées aux problématiques proposées.

## Outils et Organisation
Nous avons utilisé **PostgreSQL** (via pgAdmin) pour exécuter nos requêtes SQL, ainsi que **GitHub** pour gérer le versionning du projet. [cite_start]Nous avons travaillé en **trinôme** et chacun a participé à toutes les étapes du projet[cite: 19, 21].

## Analyse de la base de données
[cite_start]La base de données est composée de quatre tables principales : les **véhicules**, les **stations**, les **clients** et les **locations**[cite: 24]. [cite_start]Elles sont reliées entre elles grâce à des **clés primaires et étrangères**, ce qui nous a permis de travailler sur des jointures et de mieux comprendre le fonctionnement d’une base relationnelle.

## Déroulement du projet
Le projet est organisé en plusieurs quêtes :
* **Quêtes 1 et 2 :** Requêtes simples avec `SELECT`, `WHERE` et les tris.
* **Quête 3 :** Jointures entre les tables (`JOIN`).
* **Quête 4 :** Fonctions d’agrégation comme `COUNT` et `AVG`.
* **Quête 5 :** Requêtes plus complètes pour répondre à des cas concrets liés à l’activité de l’entreprise.

## Retours sur le projet
Ce projet nous a permis de mieux comprendre la logique du SQL, la structure d’une base de données et l’importance de savoir expliquer et justifier ses requêtes.
